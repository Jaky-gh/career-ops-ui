How to Use Career-Ops

              
A quick reference for reviewing opportunities and keeping the tracker current.

            

          


          

            

              
1

              

                
Check the workspace

                
The sidebar shows which career-ops checkout is connected. If the path is wrong, update settings.local.json and restart the server.

                node server.mjs
              

            


            

              
2

              

                
Review applications

                
Use Applications for the working list. Search across company, role, status, location, and notes; use filters when you want only applied roles, skips, or pipeline items.

              

            


            

              
3

              

                
Open details and reports

                
Select any row to inspect the notes panel. Use Open Posting for the job URL and View Report for the evaluation write-up linked from applications.md.

              

            


            

              
4

              

                
Use History for status checks

                
History shows tracker records sorted by date. Click a status card to jump back to Applications filtered to that status.

              

            


            

              
5

              

                
Queue new postings

                
Pipeline is for jobs that still need evaluation. Paste a posting URL and queue it, or run the job scan when you want career-ops to find candidates.

              

            


            

              
6

              

                
Run maintenance commands

                
Commands runs local scripts from the connected checkout. Start with Doctor when setup looks off, then use verify, merge, dedup, and normalize to keep data tidy.
